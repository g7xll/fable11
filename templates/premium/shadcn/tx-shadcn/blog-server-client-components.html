Server and Client Components

by shadcn

Next.js 13 introduced server and client components as part of the new App Router. This new paradigm changes how we think about rendering in React applications and provides powerful new capabilities for building modern web apps.

Server Components

Server components render on the server and send HTML to the client. They can directly access databases, file systems, and other server-side resources without exposing sensitive data to the browser. Server components have zero JavaScript bundle impact since they never run in the browser.

By default, all components in the Next.js 13 app directory are Server Components. This means you get server rendering by default without having to opt in.

// app/page.tsx
export default function Page() {
  return <h1>Hello, World!</h1>
}

Server components can fetch data directly without needing useEffect or useState. They support async/await natively:

// app/posts/page.tsx
async function getPosts() {
  const res = await fetch('https://api.example.com/posts')
  return res.json()
}

export default async function PostsPage() {
  const posts = await getPosts()

  return (
    <ul>
      {posts.map((post) => (
        <li key={post.id}>{post.title}</li>
      ))}
    </ul>
  )
}

Client Components

Client components run in the browser and can use React hooks like useState, useEffect, and useContext. They're necessary for any interactive behavior that requires client-side JavaScript.

To create a client component, add the 'use client' directive at the top of the file:

'use client'

import { useState } from 'react'

export default function Counter() {
  const [count, setCount] = useState(0)
  return (
    <button onClick={() => setCount(count + 1)}>
      Count: {count}
    </button>
  )
}

When to Use Each

You can mix server and client components in the same app. Use server components for:

  • Fetching data from databases or APIs
  • Accessing backend resources directly
  • Keeping sensitive information on the server
  • Reducing client-side JavaScript

Use client components for:

  • Adding interactivity with event listeners (onClick, onChange)
  • Using React hooks (useState, useEffect, useReducer)
  • Using browser-only APIs (localStorage, window)
  • Custom hooks that depend on state or effects

Composing Server and Client Components

One of the key patterns is being able to nest client components inside server components. You can pass server components as children or props to client components:

// This is a Server Component
import ClientComponent from './ClientComponent'
import ServerComponent from './ServerComponent'

export default function Page() {
  return (
    <ClientComponent>
      <ServerComponent />
    </ClientComponent>
  )
}

This pattern allows you to maintain the server rendering benefits while still having interactive UI elements where needed. The React team calls this "interleaving" — the ability to mix server and client components in the component tree.

Understanding when to use each type of component is crucial for building efficient Next.js 13 applications that are both performant and interactive.
