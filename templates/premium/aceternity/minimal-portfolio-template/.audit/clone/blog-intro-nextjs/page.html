Introduction to Next.js

Friday, September 15, 2023 · 8 min read

Next.js has become the go-to React framework for building production-ready web applications. It extends React with powerful features like server-side rendering, static site generation, and an intuitive file-based routing system. In this introduction, we'll cover the fundamentals and help you understand why Next.js has become so popular.

What is Next.js?

Next.js is an open-source React framework created by Vercel that enables functionality such as server-side rendering and generating static websites. It provides a production-ready foundation so you can focus on building your product instead of configuring your toolchain.

Unlike plain React, Next.js handles the complex configuration of Webpack, Babel, and other build tools for you. It also provides built-in performance optimizations like image optimization, font optimization, and automatic code splitting.

Key Features

Rendering Options

Next.js supports multiple rendering strategies, and you can even mix and match them within the same application:

  • Static Site Generation (SSG) — pages are pre-rendered at build time, resulting in fast load times and easy CDN caching
  • Server-Side Rendering (SSR) — pages are rendered on each request, ideal for dynamic data that changes frequently
  • Incremental Static Regeneration (ISR) — combines SSG with the ability to revalidate and regenerate pages after deployment
  • Client-Side Rendering (CSR) — traditional React rendering in the browser, useful for highly interactive UIs 
// Static Site Generation
export async function getStaticProps() {
  const data = await fetchData();
  return { props: { data } };
}

// Server-Side Rendering
export async function getServerSideProps(context) {
  const data = await fetchData(context.params.id);
  return { props: { data } };
}

// Incremental Static Regeneration
export async function getStaticProps() {
  return {
    props: { data },
    revalidate: 60 // seconds
  };
}

File-based Routing

Next.js uses a file-system based router. Files placed inside the pages/ (or app/ with the App Router) directory automatically become routes. This eliminates the need for manual route configuration. 

pages/
  index.js       → /
  about.js       → /about
  blog/
    index.js     → /blog
    [slug].js    → /blog/:slug
  api/
    hello.js     → /api/hello

API Routes

Next.js allows you to build your API directly inside your Next.js app. Files placed in pages/api/ become API endpoints, eliminating the need for a separate backend server for many use cases. 

// pages/api/users.js
export default function handler(req, res) {
  if (req.method === 'GET') {
    const users = db.getUsers();
    res.status(200).json(users);
  } else if (req.method === 'POST') {
    const user = db.createUser(req.body);
    res.status(201).json(user);
  }
}

Built-in Optimizations

Next.js comes with several automatic performance optimizations out of the box:

  • Image Optimization — the next/image component automatically optimizes images on demand, serving them in modern formats like WebP and AVIF
  • Font Optimization — next/font automatically self-hosts and optimizes Google Fonts with zero layout shift
  • Script Optimization — the next/script component lets you control when third-party scripts load
  • Automatic Code Splitting — each page only loads the JavaScript it needs, reducing initial bundle size

Getting Started

Creating a new Next.js project is straightforward with the CLI: 

npx create-next-app@latest my-app
cd my-app
npm run dev

This creates a new Next.js application with TypeScript support, ESLint, and Tailwind CSS configured out of the box. Visit http://localhost:3000 to see your app running.

Next.js is the right choice for most React projects. Whether you're building a blog, an e-commerce store, a SaaS application, or a marketing site, Next.js has the rendering strategy, routing capabilities, and optimizations you need to ship fast, production-quality applications.
