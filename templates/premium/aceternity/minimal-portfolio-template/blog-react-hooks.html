Mastering React Hooks

Monday, January 15, 2024 · 10 min read

React Hooks revolutionized how we write React components by allowing us to use state and other React features in functional components. Since their introduction in React 16.8, hooks have become an essential part of modern React development. This guide covers the most important hooks and how to use them effectively.

useState

The useState hook is the most fundamental hook. It lets you add state to functional components. It returns an array with the current state value and a function to update it. 

import { useState } from 'react';

function Counter() {
  const [count, setCount] = useState(0);

  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={() => setCount(c => c + 1)}>
        Increment
      </button>
    </div>
  );
}

When updating state based on the previous value, always use the functional update form (setCount(c => c + 1)) to avoid stale closure bugs. State updates from useState are asynchronous and batched in React 18.

useEffect

The useEffect hook lets you perform side effects in functional components. It runs after every render by default, but you can control when it runs using the dependency array. 

import { useState, useEffect } from 'react';

function UserProfile({ userId }) {
  const [user, setUser] = useState(null);

  useEffect(() => {
    let cancelled = false;

    async function fetchUser() {
      const data = await api.getUser(userId);
      if (!cancelled) setUser(data);
    }

    fetchUser();

    return () => {
      cancelled = true; // cleanup
    };
  }, [userId]); // re-run when userId changes

  if (!user) return <p>Loading...</p>;
  return <p>{user.name}</p>;
}

Always clean up subscriptions, timers, and async operations in the cleanup function to avoid memory leaks and state updates on unmounted components.

useCallback

The useCallback hook returns a memoized callback function. It's useful when passing callbacks to optimized child components that rely on reference equality to avoid unnecessary renders. 

import { useState, useCallback } from 'react';

function SearchBar({ onSearch }) {
  const [query, setQuery] = useState('');

  const handleSubmit = useCallback((e) => {
    e.preventDefault();
    onSearch(query);
  }, [query, onSearch]);

  return (
    <form onSubmit={handleSubmit}>
      <input
        value={query}
        onChange={e => setQuery(e.target.value)}
      />
      <button type="submit">Search</button>
    </form>
  );
}

Don't over-use useCallback. Only memoize functions when the child component is wrapped in React.memo or when the function is a dependency in another hook's dependency array.

useMemo

The useMemo hook memoizes the result of an expensive computation. It recomputes the value only when its dependencies change, preventing unnecessary recalculations on every render. 

import { useMemo } from 'react';

function FilteredList({ items, filter }) {
  const filteredItems = useMemo(() => {
    return items.filter(item =>
      item.name.toLowerCase().includes(filter.toLowerCase())
    );
  }, [items, filter]);

  return (
    <ul>
      {filteredItems.map(item => (
        <li key={item.id}>{item.name}</li>
      ))}
    </ul>
  );
}

Like useCallback, only use useMemo when the computation is genuinely expensive. Premature optimization with memoization can actually hurt performance due to the overhead of the memoization itself.

Custom Hooks

Custom hooks let you extract component logic into reusable functions. A custom hook is a JavaScript function whose name starts with "use" and that may call other hooks. 

function useLocalStorage(key, initialValue) {
  const [storedValue, setStoredValue] = useState(() => {
    try {
      const item = window.localStorage.getItem(key);
      return item ? JSON.parse(item) : initialValue;
    } catch {
      return initialValue;
    }
  });

  const setValue = useCallback((value) => {
    try {
      const valueToStore = value instanceof Function
        ? value(storedValue)
        : value;
      setStoredValue(valueToStore);
      window.localStorage.setItem(key, JSON.stringify(valueToStore));
    } catch (error) {
      console.error(error);
    }
  }, [key, storedValue]);

  return [storedValue, setValue];
}

// Usage
function ThemeToggle() {
  const [theme, setTheme] = useLocalStorage('theme', 'light');
  return (
    <button onClick={() => setTheme(t => t === 'light' ? 'dark' : 'light')}>
      Current theme: {theme}
    </button>
  );
}

Advanced Hook Patterns

useReducer for Complex State

When state logic becomes complex, useReducer is often cleaner than multiple useState calls. It follows the Redux pattern of dispatching actions to update state. 

import { useReducer } from 'react';

function reducer(state, action) {
  switch (action.type) {
    case 'INCREMENT':
      return { count: state.count + 1 };
    case 'DECREMENT':
      return { count: state.count - 1 };
    case 'RESET':
      return { count: 0 };
    default:
      throw new Error('Unknown action: ' + action.type);
  }
}

function Counter() {
  const [state, dispatch] = useReducer(reducer, { count: 0 });
  return (
    <div>
      <p>{state.count}</p>
      <button onClick={() => dispatch({ type: 'INCREMENT' })}>+</button>
      <button onClick={() => dispatch({ type: 'DECREMENT' })}>-</button>
      <button onClick={() => dispatch({ type: 'RESET' })}>Reset</button>
    </div>
  );
}

useContext for Shared State

Combine useContext with useReducer to build a lightweight state management solution without external libraries. Create a context, wrap your app with the provider, and consume it in any component. 

const ThemeContext = createContext(null);

function ThemeProvider({ children }) {
  const [theme, setTheme] = useState('light');
  return (
    <ThemeContext.Provider value={{ theme, setTheme }}>
      {children}
    </ThemeContext.Provider>
  );
}

function useTheme() {
  const ctx = useContext(ThemeContext);
  if (!ctx) throw new Error('useTheme must be used inside ThemeProvider');
  return ctx;
}

Hooks have fundamentally changed React development for the better. They make code more readable, reusable, and testable. The key is knowing which hook to use for each situation and understanding the rules — always call hooks at the top level, and always call them from React function components or custom hooks.
