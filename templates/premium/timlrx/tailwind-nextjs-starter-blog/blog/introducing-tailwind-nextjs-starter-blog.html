Introducing Tailwind Nextjs Starter Blog

Tails Azimuth

@TailsAzimuth

Looking for a performant, out of the box template, with all the best in web technology to support your blogging needs? Checkout the Tailwind Nextjs Starter Blog template.

This is a Next.js, Tailwind CSS blogging starter template. Version 2 is based on Next App directory with React Server Component and uses Contentlayer to manage markdown content.

Motivation

I wanted to port my existing blog to Nextjs and Tailwind CSS but could not find one that satisfied my needs so I decided to create one. Design is adapted from Tailwind CSS blog and leerob.io. Feel free to use this template for your own blogging purposes as well.

Features

  • Next.js with Typescript
  • Contentlayer to manage content logic
  • Easy styling customization with Tailwind 3.0 and primary color attribute
  • Near perfect lighthouse score - Performance, Accessibility, SEO, Best Practices
  • Lightweight, 85kB first load JS
  • Mobile-friendly view
  • Light and dark theme
  • Font optimization with next/font
  • Integration with Plausible, Simple Analytics, Umami, Posthog or Google Analytics
  • Structured data / JSON-LD for SEO
  • RSS, sitemap, and social sharing support
  • Support for nested routing of blog posts
  • Projects page
  • Preconfigured security headers
  • SEO friendly with canonical URLs and other meta tags

Quick Start Guide

Try installing and running locally with the following steps: 

npx degit 'timlrx/tailwind-nextjs-starter-blog'
npm install
npm run dev

Open http://localhost:3000 with your browser to see the result.

Edit the layout in app or content in data. With live reloading, the pages update automatically as you edit them.

Setting up KBar Search

By default, a local search based on kbar is used. To use it, create a data/search.json file with a list of blog posts. Alternatively, Algolia can be configured to use the Algolia search provider.

Extend / Customize

data/siteMetadata.js - contains most of the site related information which should be modified for a user's need.

data/authors/default.mdx - default author information (required). Additional authors can be added as files in data/authors.

data/projectsData.ts - data used to generate styled card on the projects page.

data/headerNavLinks.ts - navigation links.

data/logo.svg - replace with your own logo.

data/blog - replace with your own blog posts.

public/static - store assets such as images and favicons.

tailwind.config.js and css/tailwind.css - contain the tailwind stylesheet which can be modified to change the overall look and feel of the site.
