How to Integrate Chargebee with Your App

Step 1

Create your Chargebee account

Begin by signing up for a Chargebee account at chargebee.com. During the setup process, you'll configure your site name — this becomes your unique subdomain (e.g., yourcompany.chargebee.com) and is required for all subsequent API calls. Choose the plan that matches your billing volume and enable the product catalog features you intend to use: products, plans, add-ons, and coupons. Once your account is active, navigate to the Settings section to configure your default currency, tax settings, and invoice branding before connecting to external systems.

Step 2

Generate API credentials

Chargebee uses API keys for authentication. From your Chargebee dashboard, go to Settings → API Keys and Webhooks. Create a new full-access API key for your production environment, and a separate test-mode key for your staging setup. Copy both keys immediately and store them securely in your environment variable manager — Chargebee will not display full keys again after creation. Your API key is passed as the username in HTTP Basic Auth with an empty password field. Confirm access by making a test call to the /api/v2/customers endpoint; a 200 response confirms your credentials are valid.

Step 3

Connect to Ælen

In your Ælen CRM workspace, navigate to Settings → Integrations → Chargebee. Paste your Chargebee site name and API key into the configuration form. Ælen will perform a handshake verification, pulling a small batch of existing subscriptions to confirm read access. Once verified, you'll see a green confirmation badge and the sync options panel will unlock. You can choose to do an initial full sync of your existing Chargebee customer and subscription data into Ælen contact records, or opt for forward-only syncing if your Chargebee history is large and you prefer a clean start from the connection date.

Step 4

Configure webhook events

Webhooks are what keep Ælen updated in real time as billing events occur in Chargebee. In Chargebee's Settings → API Keys and Webhooks → Webhooks panel, add a new webhook endpoint pointing to your Ælen webhook URL — found in Ælen under Settings → Integrations → Chargebee → Webhook URL. Select the event types you want Ælen to receive: at minimum, enable subscription_created, subscription_renewed, subscription_cancelled, customer_created, and invoice_generated. These events trigger automatic updates to the corresponding contact records and deal stages in Ælen without requiring manual data entry. Test each webhook using Chargebee's built-in event replay tool to confirm Ælen is receiving and processing events correctly.

Step 5

Test the integration

Before going live, use Chargebee's test mode to simulate the full subscription lifecycle: create a test customer, subscribe them to a plan, renew the subscription, then cancel it. In parallel, watch Ælen's activity feed for the corresponding contact to confirm each event is being captured and mapped correctly. Verify that subscription status, MRR value, and next renewal date are appearing on the correct fields in Ælen's contact detail view. Run at least one invoice generation event and confirm it surfaces as a deal activity in the associated pipeline record. Once all five key events validate cleanly in test mode, switch your Chargebee webhook to point at your production Ælen endpoint and flip the integration toggle from Test to Live. Your Chargebee billing data will now flow into Ælen automatically.
