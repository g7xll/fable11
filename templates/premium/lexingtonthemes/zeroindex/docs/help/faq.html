Frequently Asked Questions

Answers to common questions about using and contributing to our documentation.

General Documentation Questions

How do I add a new page to the documentation?

To add a new page, create a new .mdx or .md file inside the src/content/docs/ directory. The file path becomes the URL path — for example, src/content/docs/guides/my-guide.mdx will be available at /docs/guides/my-guide. Make sure to include frontmatter with at least a title and description.

Can I use custom components inside MDX files?

Yes! MDX fully supports JSX and Astro components. You can import any component at the top of your .mdx file using a standard import statement: import MyComponent from '../../components/MyComponent.astro'. Then use it inline in your content like <MyComponent />.

How do I customize the sidebar navigation?

The sidebar is configured in src/config/navigation.ts. You can add, remove, or reorder sections and links there. Each section supports a label, an optional icon, and an array of page links. See the Sidebar documentation for the full configuration reference.

Does ZeroIndex support multiple languages?

Internationalization (i18n) is supported via Astro's built-in i18n routing. You can organize your content by locale (src/content/docs/en/, src/content/docs/fr/, etc.) and configure the supported locales in astro.config.mjs. A language switcher component is available in the src/components/ directory.

Contributing to Documentation

How do I report an error in the documentation?

The best way to report an error is to open a GitHub issue in the repository. Include the page URL, describe what's incorrect, and provide the correct information if you know it. You can also click the "Edit this page" link at the bottom of any docs page to propose a fix directly via a pull request.

What writing style should I follow when contributing?

We follow a clear, concise technical writing style. Use active voice, second person ("you"), and plain language. Avoid jargon unless it's standard in the field and explained on first use. Code examples should be minimal and runnable. See the CONTRIBUTING.md file in the repository for the full style guide.

Can I propose a new documentation section?

Absolutely. Open a GitHub issue with your proposal and label it "enhancement". Describe the topic you want covered, why it would be valuable to readers, and any existing resources you'd reference. The maintainers will review it and work with you to shape the new section before you start writing.

How long does it take for a PR to be reviewed?

Most PRs are reviewed within 2-5 business days. Typo fixes and small corrections are typically merged same-day. Larger contributions may take a week or more as they require more thorough review. If your PR hasn't received a response after a week, feel free to leave a comment to ping the team.

Technical Documentation Features

How does the full-text search work?

Search is powered by Fuse.js, a lightweight fuzzy-search library. At build time, all documentation pages are indexed into a JSON file. When you open the search modal (⌘K / Ctrl+K), the index is loaded once and all subsequent searches run entirely in the browser — no server required. Results are ranked by title match, then content match.

How is dark mode implemented?

Dark mode uses a .dark class on the <html> element combined with CSS custom properties (variables). A no-flash script runs before page render to apply the stored theme preference from localStorage before the browser paints, preventing a white flash on page load. The three available modes are: system (follows OS preference), light, and dark.

What syntax highlighter is used for code blocks?

ZeroIndex uses Shiki for syntax highlighting. Shiki runs at build time and produces static HTML — there's no runtime JS needed for highlighting. It supports hundreds of languages and themes. The default themes are github-light for light mode and github-dark for dark mode, and they switch automatically with the theme toggle.

Can I add custom fonts or change the typography?

Yes. The default font is Geist (from Vercel) loaded via Google Fonts. To change it, update the font link in your layout's <head> and change the --font-sans and --font-mono CSS custom properties in tokens.css. All type sizing uses a modular scale defined in CSS variables so you can adjust the scale globally.

About the Docs Site

What framework powers ZeroIndex?

ZeroIndex is built with Astro, a modern web framework optimized for content-driven websites. Astro ships zero JavaScript by default, resulting in fast page loads and excellent Lighthouse scores. The styling is done with Tailwind CSS, and content is written in MDX, which lets you embed interactive components inside Markdown files.

Is ZeroIndex open source?

ZeroIndex is a commercial template sold via Polar.sh. The license allows you to use it for any number of personal or client projects. You may not resell or redistribute the template itself. See the Licensing page for the full terms.

How often is ZeroIndex updated?

ZeroIndex follows Astro's release cycle with major updates shipped quarterly and patch releases as needed. All updates are backwards-compatible within a major version. You can find a full history of changes on the Changelog page.

Where can I get support if I'm stuck?

After checking this FAQ and the relevant docs pages, you can reach the team via the GitHub Discussions tab on the repository, or email support at the address listed in your purchase receipt. Community help is also available in the Astro Discord server in the #help channel.
