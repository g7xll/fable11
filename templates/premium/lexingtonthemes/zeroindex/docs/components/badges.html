Badge

Small, contextual indicators for labeling, categorizing, or status messaging.

Overview

The Badge component is a compact inline label used to communicate status, category, version, or any short contextual information at a glance. Badges are non-interactive by default and designed to sit alongside text, inside table cells, or next to headings to provide supplementary metadata without disrupting reading flow.

Each badge variant carries a distinct color signal — use them consistently so readers can build intuition around what each color means in your documentation or product UI.

Basic Usage

Import the component and render it with a variant of your choice. The default variant requires no additional props. 

---
import Badge from '@components/Badge.astro';
---

<Badge>Default</Badge>
<Badge variant="accent">Accent</Badge>
<Badge variant="success">Success</Badge>
<Badge variant="warning">Warning</Badge>
<Badge variant="error">Error</Badge>
Default Accent Success Warning Error

Props

The Badge component accepts the following props:

Prop Type Default Description
variant string "default" Visual style variant. One of default, accent, success, warning, error.
size string "md" Size of the badge. Accepts sm, md, or lg.
class string "" Additional CSS class names to apply to the badge element.

Variants

Five semantic variants cover the most common documentation and UI needs. Choose the variant that best matches the intent of the label. 

<Badge variant="default">Default</Badge>
<Badge variant="accent">Accent</Badge>
<Badge variant="success">Success</Badge>
<Badge variant="warning">Warning</Badge>
<Badge variant="error">Error</Badge>
Default Accent Success Warning Error

With Icons

Badges can include a small leading icon to reinforce meaning. Keep icons at 10×10 px so they stay proportional to the badge's compact text size. 

<Badge variant="success">
  <svg ...><!-- check icon --></svg>
  Active
</Badge>

<Badge variant="error">
  <svg ...><!-- x icon --></svg>
  Deprecated
</Badge>

<Badge variant="warning">
  <svg ...><!-- warning icon --></svg>
  Beta
</Badge>
Active Deprecated Beta

Usage Examples

Badges are one of the most versatile micro-components in a documentation site. Here are the most common places you'll reach for them:

  • Version labels — Mark an API method or prop as v2.0 to indicate when it was introduced or when it changed behavior.
  • Status in tables — A Stable or Experimental badge next to each feature row communicates maturity at a glance without adding a full column of prose.
  • Feature flags — Tag a configuration option with Deprecated to warn users before they build on it, or New to surface recent additions in a changelog-style layout.
  • Category tags — In blog index pages or component galleries, UI and Utility tags let readers filter content quickly without a full tag-cloud widget.

When using badges in body text, prefer the badge-default or badge-accent variants to avoid visual noise. Reserve badge-success, badge-warning, and badge-error for situations where status urgency genuinely matters — overusing semantic color reduces its effectiveness.
