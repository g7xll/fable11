Button

A flexible button component with multiple variants, sizes, and icon support.

Overview

The Button component is a foundational interactive element used throughout the design system. It supports four visual variants — primary, secondary, outline, and ghost — covering the full spectrum from high-emphasis calls to action to subtle utility controls. Buttons can render as either a <button> element or an <a> anchor tag depending on whether an href prop is provided, making them equally suitable for actions and navigation.

Basic Usage

Import the Button component and render it with a variant. The default variant is "primary": 

---
import Button from '@/components/Button.astro';
---

<Button variant="primary">Get Started</Button>

Props

The Button component accepts the following props:

Prop Type Default Description
variant string "primary" Visual style: "primary", "secondary", "outline", or "ghost"
size string "md" Button size: "sm", "md", or "lg"
disabled boolean false When true, disables interaction and applies reduced opacity
href string undefined When provided, renders as an <a> tag instead of <button>
icon Component undefined Optional icon component rendered alongside the button label
iconPosition string "left" Position of the icon relative to the label: "left" or "right"
class string "" Additional CSS classes to merge onto the button element

Variants

Four visual variants are available, each suited to a different level of emphasis in your interface: 

<Button variant="primary">Primary</Button>
<Button variant="secondary">Secondary</Button>
<Button variant="outline">Outline</Button>
<Button variant="ghost">Ghost</Button>

Sizes

Three sizes let you fit buttons into any layout context. The default size is "md": 

<Button variant="primary" size="sm">Small</Button>
<Button variant="primary" size="md">Medium</Button>
<Button variant="primary" size="lg">Large</Button>

With Icons

Pass an icon component to the icon prop, and control placement with iconPosition. Icons are automatically sized and aligned with the label text: 

<Button variant="primary" icon={ArrowRightIcon} iconPosition="left">
  Get Started
</Button>

<Button variant="secondary" icon={DownloadIcon} iconPosition="right">
  Download
</Button>

Disabled State

Set disabled to true to prevent interaction. The button retains its visual variant but is styled with reduced opacity and a not-allowed cursor. When rendered as an anchor, the href is removed and aria-disabled="true" is applied: 

<Button variant="primary" disabled>Disabled Primary</Button>
<Button variant="secondary" disabled>Disabled Secondary</Button>
<Button variant="outline" disabled>Disabled Outline</Button>

Link Buttons

Providing an href prop causes the component to render as an <a> element, making it suitable for page navigation while retaining all button styling. This keeps your markup semantic without duplicating styles: 

<Button variant="primary" href="/docs/getting-started/introduction">
  Read the Docs
</Button>

<Button variant="outline" href="https://github.com" target="_blank">
  View on GitHub
</Button>
Read the Docs View on GitHub

Accessibility

  • When using href, the component renders a native <a> element, ensuring correct keyboard focus and screen reader semantics for navigation links.
  • Disabled buttons retain the disabled attribute (for <button>) or aria-disabled="true" (for <a>) so assistive technologies can correctly communicate the state.
  • Icon-only buttons should always include a visible label or an aria-label attribute — do not rely solely on the icon to convey meaning.
  • The ghost variant has lower visual contrast; avoid using it in contexts where it may be difficult to distinguish from static text.
  • Ensure focus styles are preserved — do not suppress the :focus-visible ring in custom class overrides.
