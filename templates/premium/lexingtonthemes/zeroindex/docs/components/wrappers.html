Wrapper

A utility layout component for wrapping content with standard or prose-style layouts.

Overview

The Wrapper component is a layout utility that constrains content width and applies consistent horizontal padding across pages. It's the foundation for page-level layout in ZeroIndex — every content section, documentation page, and marketing block passes through some form of the Wrapper to ensure a cohesive reading width and alignment grid.

Rather than repeating max-width and margin: 0 auto declarations across every section, the Wrapper encapsulates this pattern into a single composable component. This makes it easy to maintain layout consistency as the design evolves — changing one value in the component cascades everywhere it's used.

Basic Usage

Wrap any block of content with the Wrapper component to apply the default constrained width and horizontal padding: 

<Wrapper>
  <p>Your content here</p>
</Wrapper>
Content constrained to max-width: 72rem with auto margins

Props

The Wrapper component accepts the following props to customize its layout behavior:

Prop Type Default
maxWidth string "72rem"
padding string "1.5rem"
class string ""

Prose Wrapper

For typography-rich pages like documentation articles, add a div with the .prose class inside the Wrapper. This activates the typography system — setting spacing between headings, paragraphs, code blocks, lists, and blockquotes to comfortable documentation defaults. 

<Wrapper>
  <div class="prose">
    <h1>Title</h1>
    <p>Body content</p>
  </div>
</Wrapper>

When content inside a prose area needs to break out of prose styling — for example, to render a live component demo or a data table — wrap it in a div with the .not-prose class. This resets typography styles for that element only while keeping the prose context for siblings.

Full Width

Some sections should span the full viewport width rather than being constrained by the wrapper — hero areas, full-bleed images, or sections with a distinct background color that needs to extend to the screen edges. Pass maxWidth="none" or add a class="full-width" to remove the constraint: 

<!-- Full-bleed background strip -->
<Wrapper maxWidth="none" class="bg-muted">
  <p>This section spans the full viewport width.</p>
</Wrapper>

<!-- Or via class -->
<Wrapper class="full-width">
  <img src="hero.jpg" alt="Hero" />
</Wrapper>

Common use cases for full-width wrappers include hero sections with gradient or image backgrounds, testimonial strips, feature banding sections, and full-bleed media embeds.

Nesting

Wrappers can be nested for complex layouts. A common pattern is an outer full-width wrapper that carries a background color or border, with an inner constrained wrapper that positions the actual content within the readable max-width. This separates visual styling concerns (background, border) from content layout (width, alignment). 

<!-- Outer wrapper: full-bleed dark background -->
<Wrapper maxWidth="none" class="bg-dark">
  <!-- Inner wrapper: constrain content width -->
  <Wrapper>
    <h2>Featured Section</h2>
    <p>Content stays readable while the background fills the viewport.</p>
  </Wrapper>
</Wrapper>

Avoid nesting more than two levels deep — deeply nested wrappers can make it harder to reason about layout structure. If you find yourself stacking more than two, consider whether a CSS Grid or Flexbox layout on the parent would be cleaner.
