Tabs

An accessible tab interface with buttons and panels for switching between related content.

Overview

The Tabs component organizes related content into labeled panels, only one of which is visible at a time. Tabs reduce cognitive load by letting readers focus on the section most relevant to them — common examples include switching between language variants in a code example, toggling between a preview and its source, or separating basic and advanced configuration options.

ZeroIndex ships two visual tab styles: a default underline variant that works well inside documentation prose, and a pill variant suited for standalone UI sections and dashboards.

Basic Usage

Pass an array of tab labels to the items prop and provide a matching set of content panels as slot children. 

---
import Tabs from '@components/Tabs.astro';
---

<Tabs items={["Preview", "Code", "Props"]}>
  <div slot="tab-0">
    <p>Live preview of the component goes here.</p>
  </div>
  <div slot="tab-1">
    <pre><code>...source code...</code></pre>
  </div>
  <div slot="tab-2">
    <p>Props table goes here.</p>
  </div>
</Tabs>

Props

The Tabs component accepts the following props:

Prop Type Default Description
items string[] [] Array of string labels for each tab button. The array length determines the number of panels.
defaultTab number 0 Zero-based index of the tab that should be active on first render.
class string "" Additional CSS class names applied to the root tab container element.

Interactive Example

Click each tab button below to switch panels. The active tab receives a bottom border highlight; inactive tabs appear muted.

This is the preview panel. It shows a live render of the component.

<Tabs items= { ["Preview", "Code", "Props"]} />

Items: string[] — array of tab labels.
defaultTab: number — index of initially active tab.

Variants

ZeroIndex provides two visual tab styles to fit different contexts:

  • Underline (default) — Tab buttons sit directly above the content area with a bottom border indicator on the active item. This is the standard pattern for documentation pages and works well inside a prose container.
  • Pill — The active tab gets a raised card background inside a pill-shaped track. This style suits settings panels, dashboards, and any surface where tabs are visually prominent.

The pill variant example:

<Tabs items={["Overview", "API", "Examples"]} variant="pill" />

Accessibility

The Tabs component is built with ARIA semantics to ensure full keyboard accessibility:

  • Arrow key navigation — Pressing ← and → while focus is inside the tab list moves focus between tab buttons and activates the focused tab.
  • role="tablist" — Applied to the container element that wraps all tab buttons, signaling to assistive technology that these elements function as a tab list.
  • role="tab" — Applied to each individual tab button so screen readers announce them as tab controls rather than plain buttons.
  • aria-selected — Set to "true" on the currently active tab button and "false" on all others, communicating selection state without relying on visual styling alone.
  • role="tabpanel" — Applied to each content panel, paired with an aria-labelledby attribute pointing back to its controlling tab button for a complete relationship declaration.
  • Home / End keys — Jump focus directly to the first or last tab button respectively.
