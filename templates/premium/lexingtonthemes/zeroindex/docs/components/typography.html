Typography

A flexible typography component for semantic text with custom sizing and styling in ZeroIndex.

Overview

ZeroIndex uses a prose-based typography system built on semantic HTML, ensuring consistent visual hierarchy, readability, and dark mode compatibility across all documentation pages. Rather than relying on utility classes scattered throughout markup, typography is expressed through meaningful HTML elements — h1 through h4, p, ul, ol, blockquote, and code — that carry both semantic meaning and visual intent.

The system is built on CSS custom properties for easy theming. Every color, spacing value, and font size references a token from tokens.css, so a single change propagates through the entire design. Both light and dark modes are handled automatically via the .dark class on the document root.

Headings

Four heading levels are available, each with a distinct size, weight, and line height tuned for documentation readability. Use them to establish a clear visual hierarchy within a page.

Heading 1 — Page Title

Heading 2 — Section Title

Heading 3 — Subsection

Heading 4 — Minor Title 

/* Heading sizes */
h1 { font-size: 2rem;    font-weight: 700; line-height: 1.2; }
h2 { font-size: 1.5rem;  font-weight: 600; line-height: 1.3; }
h3 { font-size: 1.25rem; font-weight: 600; line-height: 1.4; }
h4 { font-size: 1rem;    font-weight: 600; line-height: 1.5; }

Body Text

Paragraph text comes in two sizes: the default body size at 1rem and a secondary size at 0.875rem. Secondary text uses a muted color token to create visual contrast without relying on size alone.

This is body text at the default size (1rem / 16px). It uses a comfortable line height of 1.75 for optimal readability in long-form documentation content.

This is secondary body text at 0.875rem. It appears slightly smaller and uses a muted color, suitable for captions, meta information, or supporting text.

Code

Inline code is styled with a monospace font, a subtle background, and a fine border so it stands out cleanly within prose without breaking reading flow. Use it to reference file names, variable names, CLI commands, or short snippets inline.

Use npm install to add dependencies, or run npm run dev to start the development server.

Links

Links within prose inherit the accent color token and include a subtle underline with a text-underline-offset to keep the underline clear of descenders. This makes links easy to spot without distracting from surrounding text.

Read the documentation for a full API reference, or visit the GitHub repository to contribute.

Lists

Both ordered and unordered lists are supported. List items use the same font size and line height as body text, with standard browser indentation preserved. Use unordered lists for non-sequential items and ordered lists when sequence or priority matters.

Blockquotes

Blockquotes use a left accent border and a muted background to visually separate quoted content from surrounding prose. An optional footer element below the quote text can credit the source or author.
