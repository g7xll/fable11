Alert

Contextual alert banners for feedback, messaging, or inline warnings.

Overview

The Alert component provides contextual feedback messages for user actions or system states. It supports multiple semantic variants — warning, error, info, success, and neutral — each communicating a different level of urgency or meaning. Alerts can include titles, descriptions, lists of items, inline links, and an optional dismiss button.

Basic Usage

Import and use the Alert component with at minimum a variant and title: 

---
import Alert from '@/components/Alert.astro';
---

<Alert
  variant="warning"
  title="Heads up!"
  description="This is a basic warning alert."
/>

Heads up!

This is a basic warning alert.

Props

The Alert component accepts the following props:

Prop Type Default Description
variant string "neutral" Semantic variant: "warning", "error", "info", "success", or "neutral"
title string undefined Bold heading text displayed at the top of the alert
description string undefined Supporting body text shown beneath the title
list string[] [] Array of strings rendered as a bulleted list inside the alert
links { label: string; href: string }[] [] Array of link objects rendered as inline anchor elements
dismissible boolean false When true, renders a close button that removes the alert from the DOM
icon Component | null undefined Override the default variant icon, or pass null to remove it
role string "alert" ARIA role for the alert container; use "status" for non-urgent messages

Variants

Five semantic variants are available. Choose the one that best matches the context of your message: 

<Alert variant="warning" title="Warning Alert" description="Something needs your attention before continuing." />
<Alert variant="error"   title="Error Alert"   description="An unexpected error occurred. Please try again." />
<Alert variant="info"    title="Info Alert"    description="Here is some information you might find useful." />
<Alert variant="success" title="Success Alert" description="Your changes have been saved successfully." />
<Alert variant="neutral" title="Neutral Alert" description="This is a general-purpose informational message." />

Lists

Use the list prop to display a bulleted list of items inside the alert, useful for requirements or multi-step notices: 

<Alert
  variant="info"
  title="Requirements"
  description="This feature requires the following dependencies:"
  list={["Node.js v18+", "Astro v4", "Tailwind CSS 3.4"]}
/>

Requirements

This feature requires the following dependencies:
  • Node.js v18+
  • Astro v4
  • Tailwind CSS 3.4

Links

The links prop accepts an array of { label, href } objects rendered as inline links beneath the description: 

<Alert
  variant="success"
  title="New Feature"
  description="Version 2.0 is now available with improved performance."
  links={[
    { label: "View Docs", href: "/docs" },
    { label: "Changelog", href: "/changelog" }
  ]}
/>

New Feature

Version 2.0 is now available with improved performance.
View Docs Changelog

Dismissible

Setting dismissible to true adds an ✕ button in the top-right corner of the alert. Clicking it removes the alert from the DOM with a fade animation: 

<Alert
  variant="info"
  title="Heads Up!"
  description="You can dismiss this alert by clicking the close button."
  dismissible={true}
/>

Heads Up!

You can dismiss this alert by clicking the close button.

Custom Icon

Each variant comes with a default icon. You can override it by passing any icon component via the icon prop, or set icon= { null} to remove it entirely: 

---
import Alert from '@/components/Alert.astro';
import BellIcon from '@/icons/BellIcon.astro';
---

<!-- Override with a custom icon -->
<Alert
  variant="info"
  title="Custom Icon"
  description="This alert uses a custom bell icon."
  icon={BellIcon}
/>

<!-- Remove the icon entirely -->
<Alert
  variant="warning"
  title="No Icon"
  description="This alert renders without any icon."
  icon={null}
/>

Accessibility

  • The alert container uses role="alert" by default, which announces the message to screen readers immediately.
  • For less urgent messages, consider role="status" or aria-live="polite" to avoid interrupting the user.
  • The dismiss button includes an aria-label="Dismiss" attribute for screen reader clarity.
  • Color alone is never the only indicator of meaning — icons and text labels reinforce the semantic variant.

Best Practices

  • Use warning for important cautions that don't block the user but require attention.
  • Use error for failures, validation errors, or destructive action confirmations.
  • Use info for neutral informational notices and tips.
  • Use success to confirm completed actions or positive outcomes.
  • Keep titles short and actionable — users scan alerts quickly.
  • Avoid stacking multiple alerts; combine related messages where possible.
