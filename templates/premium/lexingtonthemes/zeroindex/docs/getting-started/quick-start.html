Quick Start Guide

Get up and running in just a few minutes with this quick start guide.

Installation

The easiest way to create a new ZeroIndex project is with the Astro CLI. Run the following command in your terminal:

bash 
npm create astro@latest -- --template zeroindex

The CLI will prompt you to choose a project name and whether to install dependencies. Once complete, navigate into your new project directory:

bash 
cd my-docs
npm install

Project Structure

After installation, your project will have the following structure:

text 
my-docs/
├── public/
│   └── favicon.svg
├── src/
│   ├── components/
│   │   ├── Sidebar.astro
│   │   └── Header.astro
│   ├── content/
│   │   └── docs/
│   │       ├── getting-started/
│   │       │   ├── introduction.mdx
│   │       │   └── quick-start.mdx
│   │       └── components/
│   ├── layouts/
│   │   └── DocsLayout.astro
│   └── pages/
│       └── docs/
│           └── [...slug].astro
├── astro.config.mjs
├── tailwind.config.mjs
└── package.json

The key directories to understand are:

  • src/content/docs/ — All your Markdown and MDX documentation files live here
  • src/components/ — Reusable Astro components for the documentation layout
  • src/layouts/ — Page layout templates
  • public/— Static assets served directly

Writing Your First Page

Documentation pages are written in MDX, a superset of Markdown that supports JSX components. Create a new file at src/content/docs/my-first-page.mdx:

mdx 
---
title: My First Page
description: This is my first documentation page.
---

# Welcome

This is my first ZeroIndex documentation page. You can use **Markdown** here,
including [links](https://astro.build), lists, and code blocks.

```js
console.log('Hello, ZeroIndex!')
```

The frontmatter block at the top (between the --- lines) defines metadata for the page. The title and description fields are used in the page header and for SEO.

Running the Dev Server

Start the local development server with:

bash 
npm run dev

Open http://localhost:4321 in your browser. Changes to your documentation files will be reflected immediately thanks to Astro's fast HMR.

Port in Use?

If port 4321 is taken, Astro will automatically try the next available port and show you the URL in the terminal.

Deploying

When you're ready to publish, build the static output with:

bash 
npm run build

This generates a dist/ folder with the static HTML output. You can deploy this to any static hosting provider:

  • Vercel — Connect your repo and deploy instantly
  • Netlify — Drag and drop the dist/ folder or connect via Git
  • GitHub Pages — Use the Astro GitHub Pages action
  • Cloudflare Pages — Zero-config deployment with global CDN

Preview your production build locally before deploying:

bash 
npm run preview
