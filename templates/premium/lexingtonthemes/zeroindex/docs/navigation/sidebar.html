Sidebar

Understanding the sidebar navigation component in ZeroIndex.

Overview

The Sidebar is the primary navigation component in ZeroIndex docs. It renders on the left side of the page, remains fixed on desktop viewports, and slides in as a full-height overlay on mobile. The sidebar contains grouped accordion sections for hierarchical navigation and flat links for top-level pages.

Its structure is designed to scale — you can add as many accordion groups and flat links as your documentation requires without breaking the layout. The sidebar also houses the logo, the "Buy" CTA in the footer, and a close button visible only on mobile.

Structure

The sidebar is composed of three distinct regions stacked vertically inside a fixed-position container:

  • sidebar-header — contains the logo link and the mobile close button
  • sidebar-nav — the scrollable navigation area holding accordion sections and flat links
  • sidebar-footer — pinned to the bottom, contains the buy/CTA button 
<aside id="sidebar" class="sidebar">
  <div class="sidebar-inner">

    <!-- Logo + mobile close button -->
    <div class="sidebar-header">
      <a href="/" class="sidebar-logo">...</a>
      <button id="sidebar-close" class="sidebar-close">...</button>
    </div>

    <!-- Scrollable navigation -->
    <nav class="sidebar-nav">
      <!-- Accordion sections -->
      <div class="accordion-section">...</div>

      <!-- Flat top-level links -->
      <a href="/integrations" class="sidebar-flat-link">Integrations</a>
    </nav>

    <!-- CTA pinned to bottom -->
    <div class="sidebar-footer">
      <a href="#" class="btn-buy">Buy ZeroIndex</a>
    </div>

  </div>
</aside>

Accordion Sections

Navigation items are grouped using accordion sections. Each section has a trigger button that toggles the visibility of its child links with a smooth height and opacity transition. Accordion sections are well-suited for organizing large documentation trees into logical topic groups.

Each accordion section follows this structure:

<div class="accordion-section" id="section-getting-started">
  <div class="accordion-section-inner">
    <div class="accordion-bg opacity-0" data-background></div>
    <button class="accordion-trigger" aria-expanded="false" data-accordion-trigger>
      Getting Started
      <svg>...chevron icon...</svg>
    </button>
    <div class="accordion-content" style="height:0;opacity:0;transform:translateY(-0.5rem)">
      <nav class="accordion-nav">
        <a href="/docs/getting-started/introduction.html">Introduction</a>
        <a href="/docs/getting-started/quick-start.html">Quick Start Guide</a>
      </nav>
    </div>
  </div>
</div>

The data-accordion-trigger attribute is the hook used by main.js to attach the toggle behavior. The data-background div provides a subtle animated background that appears when the section is expanded. The initial style on accordion-content ensures the closed state is set before JavaScript runs, preventing a flash of expanded content on load.

Flat Links

Flat links appear below the accordion sections and are used for top-level pages that don't belong to a collapsible group. Common examples include Integrations, Changelog, and Licensing pages. 

<a href="../../integrations.html" class="sidebar-flat-link">Integrations</a>
<a href="../../changelog.html" class="sidebar-flat-link">Changelog</a>
<a href="../../infopages/licensing.html" class="sidebar-flat-link">Licensing</a>

The sidebar-flat-link class styles these as full-width clickable rows with the same hover state as accordion links, but without any indent or grouping chrome.

Mobile Behavior

On mobile viewports, the sidebar is hidden by default using transform: translateX(-100%). It becomes visible when the user taps the hamburger button (#sidebar-open) in the docs header. A semi-transparent backdrop overlay is shown behind the sidebar so users can dismiss it by tapping outside.

The sidebar can be closed in two ways on mobile:

  • Tapping the #sidebar-close button inside the sidebar header
  • Tapping the #sidebar-backdrop overlay

Mobile tip: The sidebar is toggled via the hamburger button in the docs header. On desktop (≥1024px), the sidebar is always visible and the toggle button is hidden.

Customization

The sidebar is straightforward to customize. Here are the most common modifications:

  • Adding an accordion section — Copy an existing accordion-section block, give it a unique id, update the trigger label, and add the appropriate links inside accordion-nav.
  • Adding flat links — Append a new <a class="sidebar-flat-link"> element anywhere inside sidebar-nav, after the accordion sections.
  • Changing the logo — Replace the SVG and text inside the sidebar-logo anchor. Keep the sidebar-logo class so spacing and alignment are preserved.
  • Updating the buy button URL — Change the href on the btn-buy anchor inside sidebar-footer. You can also swap the label text or remove the footer entirely if no CTA is needed.
