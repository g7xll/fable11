Sidebar Links

Understanding and configuring sidebar accordion navigation links in ZeroIndex.

Overview

Sidebar links are the individual anchor elements that live inside each accordion section's accordion-nav. They are the actual navigation items users click to move between documentation pages. Each link maps directly to one HTML file in the docs tree, and the currently active link is highlighted to help users understand where they are in the documentation.

Link Structure

A sidebar link is a plain <a> element placed inside an accordion-nav. The styling — padding, font size, border radius, and hover/active states — is applied entirely via CSS classes on the parent container. No per-link classes are needed for default behavior:

Active State

The active state is determined at page load by main.js. It reads window.location.pathname and compares it against each link's href attribute. When a match is found, the active class is applied to that anchor element and the parent accordion section is automatically opened so the active link is visible without any user interaction. 

// Simplified logic from main.js
const links = document.querySelectorAll('.accordion-nav a');
links.forEach(link => {
  if (link.href === window.location.href) {
    link.classList.add('active');
    // Open the parent accordion section
    const section = link.closest('.accordion-section');
    openAccordion(section);
  }
});

Because this runs on every page load, you never need to manually add or remove the active class in your HTML — just keep all links pointing to the correct paths.

Adding Links

To add a new link to an existing accordion section, insert a new <a> element inside the target accordion-nav: 

<nav class="accordion-nav">
  <a href="../../docs/getting-started/introduction.html">Introduction</a>
  <a href="../../docs/getting-started/quick-start.html">Quick Start Guide</a>
  <!-- New link -->
  <a href="../../docs/getting-started/configuration.html">Configuration</a>
</nav>

Remember to update the link in every HTML file across the docs, or the new link will be missing from pages that haven't been updated. If you're working with a build system or templating engine, extract the sidebar into a shared partial to avoid this overhead.

External Links

Sidebar links fully support external URLs. When linking to an outside resource, always add target="_blank" and rel="noopener noreferrer" for security and user experience: 

<nav class="accordion-nav">
  <a href="https://github.com/your-org/your-repo" target="_blank" rel="noopener noreferrer">
    GitHub Repository
  </a>
</nav>

External links will not receive the active class since their href won't match the current page's window.location.href.

Mobile Considerations

On mobile viewports the sidebar is displayed as an overlay panel. Tapping any link inside an accordion-nav navigates to the target page, which causes the sidebar to close automatically as part of the page transition. For in-page links (anchors pointing to #heading-id), main.js listens for clicks on accordion nav links and closes the sidebar before scrolling, ensuring the content is not obscured.

Ordering and Grouping

How you organize links has a meaningful impact on how quickly users find what they're looking for. A few guidelines that work well for technical documentation:

  • Workflow progression — Order links from conceptual to practical to reference. Start with overview/introduction pages, move to guides, and end with API or configuration references.
  • Group by theme — Put pages that share a common concept or feature area into the same accordion section. Avoid mixing unrelated topics in a single group.
  • Keep groups manageable — Aim for 4–8 links per accordion section. Fewer than 4 rarely justifies a collapsible group; more than 8 suggests the section should be split or the content reorganized.
  • Alphabetical ordering — Alphabetical sorting works well for reference sections (like component lists) where users know what they're looking for. For guides and tutorials, prefer narrative order instead.
