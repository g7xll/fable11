> A SELF-CONTAINED, PIXEL-FAITHFUL REPRODUCTION OF THE MATTERHAUS ARCHITECTURE THEME — EVERY PAGE, EVERY LAYOUT, HOVER STATES, AND INTERACTIONS — AS PLAIN HTML/CSS/JS.
> REFERENCE: `https://lexingtonthemes.com/viewports/matterhaus`

## SUMMARY

MATTERHAUS IS AN ARCHITECTURE / STUDIO THEME FOR ASTRO & TAILWIND CSS BY LEXINGTON THEMES. IT FEATURES A CLEAN, EDITORIAL SPLIT-LAYOUT DESIGN ACROSS ALL PAGES: A FIXED FULL-WIDTH HEADER WITH MIX-BLEND-DIFFERENCE NAV (WHITE TEXT ON ANY BACKGROUND), A CONSISTENT TWO-COLUMN SPLIT (LEFT SIDE = STICKY HEADING / INFO, RIGHT SIDE = SCROLLABLE CONTENT OR FULL-HEIGHT IMAGE), AND A FULLY FUNCTIONAL SEARCH MODAL POWERED BY FUSE.JS. THE PALETTE IS STRICTLY MONOCHROMATIC (GREYSCALE BASE SCALE) WITH AN OPTIONAL ACCENT (OKLCH BLUE/PURPLE). NO DARK MODE. LIGHT-ONLY.

## LAYOUT & STRUCTURE

### SHARED CHROME

**HEADER**: FIXED, FULL-WIDTH, Z-50, `mix-blend-difference` — WHITE LOGO TEXT "MATTERHAUS" LEFT, HAMBURGER "MENU" BUTTON RIGHT. ON MOBILE, HEADER HEIGHT = ~36PX.

**MOBILE NAV**: POSITIONED BELOW HEADER (`top: 36px`), FIXED FULL-WIDTH, Z-40, `bg-base-100`, SHOWS LIST OF PAGE LINKS EACH WITH RIGHT-ARROW ICON, PLUS SEARCH BUTTON AND "BUY" BUTTON. TOGGLED BY HAMBURGER BUTTON.

**SEARCH MODAL**: FULL-OVERLAY (`fixed inset-0 bg-black/10 backdrop-blur-sm`), BACKDROP-CLICK TO DISMISS, `<input>` + FUSE.JS FUZZY SEARCH ACROSS ALL CONTENT (POSTS, PROJECTS, SERVICES, TEAM, CAREERS, LEGAL). KEYBOARD: `/` OR CMD+K OPENS; ESC CLOSES.

### PAGES

1. **HOME** (`index.html`) — HERO SECTION: LEFT HALF = H1 HEADLINE + ADDRESS/CONTACT DETAILS AT BOTTOM (`lg:h-dvh` FLEX-COL `justify-between`); RIGHT HALF = FULL-HEIGHT COVER IMAGE (`object-cover h-dvh`).

2. **PROJECTS** (`projects/index.html`) — LEFT = STICKY H1 + DESCRIPTION; RIGHT = SCROLLABLE LIST OF 10 PROJECT ARTICLES, EACH WITH COVER IMAGE (FULL-WIDTH), TITLE, METADATA (YEAR, LOCATION), HOVER: TEXT TRANSITIONS TO DARKER SHADE.

3. **PROJECT DETAIL** (`projects/[slug].html` × 10) — LEFT = STICKY: TITLE, YEAR, TYPE, AREA, CLIENT, LOCATION + THUMBNAIL GRID (3-COL); RIGHT = FULL PROJECT IMAGES.

4. **SERVICES** (`services/index.html`) — LEFT = STICKY H1 + DESCRIPTION + SERVICES LIST (TEXT LINKS WITH ARROWS); RIGHT = EMPTY / IMAGE OR ADDITIONAL INFO.

5. **SERVICE DETAIL** (`services/[slug].html` × 8) — LEFT = STICKY HEADING + DESCRIPTION; RIGHT = SERVICE IMAGE + RICH PROSE CONTENT.

6. **STUDIO** (`studio/index.html`) — SPLIT HERO: LEFT = H1 + DESCRIPTION + 2-COL BODY TEXT; RIGHT = FULL-HEIGHT STUDIO IMAGE.

7. **CONTACT** (`contact/index.html`) — LEFT = H1 + ADDRESS BLOCK + PHONE/EMAIL + SOCIAL LINKS; RIGHT = FULL-HEIGHT CONTACT IMAGE.

8. **CAREERS** (`careers/index.html`) — LEFT = STICKY H1 + DESCRIPTION + CAREERS LIST (TEXT LINKS); RIGHT = (CAREERS RIGHT SIDE CONTENT).

9. **CAREER DETAIL** (`careers/[slug].html` × 5) — LEFT = TITLE + LOCATION/TYPE/SALARY + "APPLY NOW" LINK; RIGHT = JOB DESCRIPTION PROSE.

10. **BLOG** (`blog/index.html`) — SPLIT: LEFT = TITLE "JOURNAL"; RIGHT = BLOG ARTICLE LIST (TITLE + EXCERPT + DATE ROWS).

11. **BLOG POST** (`blog/posts/[n].html` × 4) — LEFT = DATE/AUTHOR + H1 + DESCRIPTION + PROSE; RIGHT = POST HERO IMAGE.
