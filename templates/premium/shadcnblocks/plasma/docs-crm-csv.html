CRM & CSV

Learn how to manage and import CRM data using CSV files for efficient data processing.

Overview

Plasma provides powerful tools for managing customer relationship data through CSV files and direct CRM integrations.

Prepare Your Data

Ensure your CSV files follow the correct format. Example CSV structure: 

id,name,email,company,phone,status,created
1,John Smith,john@acme.com,Acme Corp,+1-555-0100,active,2024-01-15
2,Jane Doe,jane@globex.com,Globex,+1-555-0101,prospect,2024-02-03

Import Data

Use Plasma's import functionality:

import { crm } from 'plasma';

const result = await crm.import('customers.csv', {
  mapping: {
    'Customer Name': 'name',
    'Email Address': 'email',
    'Phone Number': 'phone',
  },
  validation: true,
  batchSize: 1000,
});

Validate & Process

Review imported data and handle errors:

if (result.errors.length > 0) {
  console.log('Import errors:', result.errors);
}

console.log(`Imported ${result.successful} records`);

Data Formats

Required fields for customer data:

Column Type Notes
id string | number Unique record identifier
name string Customer full name
email string Valid email address
phone? string Optional contact number
company? string Associated company
status? 'active' | 'inactive' | 'prospect' Lifecycle stage

Data Validation

Ensure data quality with built-in validation:

import { validate } from 'plasma';

const schema = {
  email: validate.email().required(),
  phone: validate.phone().optional(),
  status: validate.enum(['active', 'inactive', 'prospect']),
};

const validated = await crm.validate(customers, schema);

Export Features

Export to multiple formats:

// Export filtered customer data
await crm.export('active-customers.csv', {
  filter: { status: 'active' },
  format: 'csv',
});

Batch Processing

Handle large datasets efficiently:

const batchProcessor = crm.createProcessor({
  batchSize: 5000,
  concurrency: 4,
});

await batchProcessor.process('large-dataset.csv', async (batch) => {
  await db.insert(batch);
});

Best Practices

Performance Tip: Use batch processing for files larger than 10,000 records to maintain optimal performance.
  • Validate data before import
  • Use consistent field naming
  • Include unique identifiers
  • Implement error handling
  • Monitor import progress
  • Backup data before bulk operations
