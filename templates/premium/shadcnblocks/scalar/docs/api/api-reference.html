API Reference

Read and write content through Scalar's REST and GraphQL APIs, with consistent JSON responses across every model.

Base URL

All REST requests are made to your project's content API, scoped by version:

base url
https://api.scalar.dev/v1

Replace { model} in the paths below with the API identifier of one of your content models, such as articles or authors.

Endpoints

The REST API exposes a predictable set of resource endpoints for every content model:

Method Endpoint Description
GET /v1/content/ { model} List entries for a model with pagination and filtering.
GET /v1/content/ { model}/ { id} Retrieve a single entry by its unique id.
POST /v1/content/ { model} Create a new entry for the model.
PATCH /v1/content/ { model}/ { id} Update fields on an existing entry.
DELETE /v1/content/ { model}/ { id} Delete an entry permanently.

Request example

Fetch a page of published articles, passing your API token as a bearer credential:

terminal
# list published articles curl https://api.scalar.dev/v1/content/articles?status=published \ -H "Authorization: Bearer $SCALAR_TOKEN" \ -H "Accept: application/json"

Response format

List endpoints return a data array alongside a meta object describing pagination:

json
{ "data": [ { "id": "ent_a1b2c3", "model": "articles", "fields": { "title": "Hello World", "status": "published" } , "updatedAt": "2026-06-22T10:30:00Z" } ], "meta": { "page": 1, "pageSize": 20, "total": 87 } }

GraphQL endpoint

Prefer GraphQL? Send queries to the single /v1/graphql endpoint to fetch exactly the fields you need:

graphql
query PublishedArticles { articles(where: { status: PUBLISHED }, first: 20) { nodes { id title author { name } updatedAt } } }

Rate limits

Requests are rate limited per API token. The current usage is returned on every response in the X-RateLimit-Remaining header.

The default limit is 1,000 requests per minute. Exceeding it returns a 429 Too Many Requests response — back off and retry after the seconds given in the Retry-After header.
