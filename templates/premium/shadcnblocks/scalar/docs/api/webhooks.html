Webhooks

Subscribe to content events and receive real-time HTTP callbacks whenever entries change in your Scalar project.

Overview

Webhooks let Scalar push events to your own services the moment they happen, so you can rebuild a static site, invalidate a cache, or notify a Slack channel without polling the API. Each webhook is a URL that Scalar sends a signed POST request to whenever a subscribed event fires.

Every delivery includes a JSON payload describing the event and an X-Scalar-Signature header you can use to verify authenticity. Failed deliveries are retried with exponential backoff for up to 24 hours.

Registering a webhook

Create a webhook by sending a POST request to the webhooks endpoint with the target URL and the events you want to subscribe to:

terminal
# register a webhook curl -X POST https://api.scalar.dev/v1/webhooks \ -H "Authorization: Bearer $SCALAR_API_KEY" \ -H "Content-Type: application/json" \ -d ' { "url": "https://example.com/hooks/scalar", "events": ["entry.published", "entry.deleted"] }'

The response returns the webhook id and a generated secret used to sign every delivery. Store the secret securely — it is only shown once.

Event types

Subscribe to any combination of the following events when registering a webhook:

Event Description
entry.created A new entry was created in any model.
entry.updated An existing entry's fields were changed and saved.
entry.published A draft entry transitioned to the published state.
entry.deleted An entry was permanently removed from a model.

Payload example

Every delivery sends a JSON body with the event name, the affected model, and the entry data at the time of the event:

application/json
{ "event": "entry.published", "model": "blog_post", "data": { "id": "ent_8f3a21", "title": "Introducing Webhooks", "slug": "introducing-webhooks", "status": "published", "publishedAt": "2026-06-23T10:14:00Z" } }

Verifying signatures

Scalar signs each request with an HMAC SHA-256 digest of the raw request body using your webhook secret. Compute the same digest on your end and compare it with the X-Scalar-Signature header before trusting the payload:

verify.js
import crypto from "node:crypto"; function verify(rawBody, signature, secret) { const expected = crypto .createHmac("sha256", secret) .update(rawBody) .digest("hex"); return crypto.timingSafeEqual( Buffer.from(expected), Buffer.from(signature) ); }
Always verify against the raw request body before parsing it as JSON. Re-serializing the parsed object can change byte order and produce a mismatched signature.
