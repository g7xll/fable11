Tutorials

Follow these end-to-end, step-by-step tutorials to build real features with Scalar CMS — from your first blog to a fully deployed site.

Build a blog

This tutorial walks you through modeling content, querying it, and rendering posts on a public page.

  1. 1Create a Post collection. In Scalar Studio, add a collection with title, slug, body, and publishedAt fields.
  2. 2Add a few entries. Write two or three draft posts and mark one as published so you have data to query.
  3. 3Query the published posts. Use the SDK to fetch entries filtered by status.
  4. 4Render the list. Map the results to a page and link each post to its slug route.
posts.ts
import { scalar } from "@scalar/sdk" const posts = await scalar.collection("posts").find( { where: { status: "published" } , sort: "-publishedAt", })
Keep drafts out of production by always filtering on status: "published" in your public queries.

Add authentication

Protect the admin panel and gate content behind signed-in users.

  1. 1Enable the auth plugin. Add @scalar/auth to your config and choose an email or OAuth provider.
  2. 2Define roles. Map admin, editor, and viewer roles to collection permissions.
  3. 3Wrap protected routes. Use the requireUser guard to redirect unauthenticated requests.
scalar.config.ts
import { auth } from "@scalar/auth" export default defineConfig( { plugins: [ auth( { provider: "email", roles: ["admin", "editor"] }), ], })

Image uploads

Let editors upload media and serve optimized images from your storage backend.

  1. 1Configure a storage adapter. Point Scalar at S3, R2, or the local disk for development.
  2. 2Add a media field. Attach an image field to your Post collection.
  3. 3Request transforms. Resize and format images on the fly with query params.
terminal
# install the storage adapter npm install @scalar/storage-s3

Deploy your site

Ship your project to production with a managed database and CDN-backed assets.

  1. 1Set environment variables. Provide DATABASE_URL and your storage credentials in the host dashboard.
  2. 2Run migrations. Apply your schema before the first boot.
  3. 3Build and start. Compile assets and launch the production server.
terminal
# apply migrations, then build npx scalar migrate deploy npm run build npm run start
