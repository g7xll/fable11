Guides

A practical tour of working with Scalar CMS — from modeling content to querying, mutating, and shipping localized, production-ready data.

Modeling content

Everything in Scalar starts with a schema. A schema describes the shape of your content as a set of collections, each with typed fields. Define collections in scalar.config.ts and Scalar generates the database tables, the admin UI, and a fully typed client for you.

  • Use singular field names and plural collection names for consistency.
  • Mark fields required only when an empty value would break rendering.
  • Prefer references over duplicated data to keep content in sync.
Schema changes are tracked as migrations. Run npx scalar migrate after editing a collection so your database stays in lockstep with the config.

Querying data

The generated client gives you a single query method that is fully typed against your schema. Select only the fields you need to keep payloads small:

app/page.ts
import { client } from "@scalar/sdk"; // fetch the latest published posts const posts = await client.query("posts", { where: { status: "published" }, select: ["title", "slug", "publishedAt"], orderBy: { publishedAt: "desc" }, limit: 10, });

Queries are resolved at the edge and cached automatically. Pass { cache: "no-store" } when you need always-fresh data.

Mutations

Create, update, and delete content with the matching client methods. Every mutation is validated against your schema before it touches the database:

app/actions.ts
import { client } from "@scalar/sdk"; // create a draft post const post = await client.create("posts", { title: "Hello, Scalar", status: "draft", }); // publish it later await client.update("posts", post.id, { status: "published" });

Wrap related writes in client.transaction(...) to keep them atomic.

Localization

Mark any field as localized: true and Scalar stores a value per locale. The client resolves the right translation based on the locale you pass:

app/page.ts
// fetch French content with an English fallback const page = await client.query("pages", { locale: "fr", fallbackLocale: "en", });

Locales are configured once in scalar.config.ts and surfaced as a language switcher in the admin panel automatically.

Best practices

A few habits keep Scalar projects fast and maintainable as they grow:

  • Select explicit fields instead of fetching whole documents.
  • Co-locate queries with the components that render them.
  • Use references and relations rather than copying content between collections.
  • Run migrations in CI so schema and database never drift apart.
  • Lean on edge caching, and reach for no-store only when freshness truly matters.
