How Scalar generates GraphQL APIs instantly

Learn how Scalar turns your content models into ready-to-query GraphQL endpoints — no backend setup required.

Fedir Davydov · 5 min read

Every Scalar project ships with a fully typed GraphQL API the moment you define your first model. No resolvers to write, no servers to deploy — just query your content. Here's how it works.

From model to schema

When you push a model, Scalar generates a corresponding GraphQL type, query fields, and filter inputs. A model named blogPost becomes a blogPost single query and an allBlogPosts collection query automatically.

query.graphql
query { allBlogPosts(filter: { published: true }) { title author { name } publishedAt } }

Relationships resolve for free

Reference fields become nested objects in GraphQL. Query an author through a post in a single round trip — Scalar batches and resolves the joins for you.

  • Automatic batching avoids the N+1 problem.
  • Depth limits protect your project from runaway queries.
  • Field-level permissions are enforced inside the resolver.

Using the generated client

Pair the API with the typed SDK and every query is checked against your schema at build time:

main.ts
const { allBlogPosts } = await client.query({ model: "blogPost", include: ["author"], });

Built-in playground

Every project includes an interactive GraphQL explorer with live schema docs, so editors and developers can prototype queries without leaving the dashboard.

Performance and caching

Responses are cached at the edge and invalidated automatically when content changes, so your published queries stay fast worldwide.

Wrap-up

Scalar removes the boilerplate between your content model and a production GraphQL API. Define your data once and query it everywhere — instantly.
