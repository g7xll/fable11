Overview

Complete guide to Scalar CMS — the modern headless content management system.

Introduction

Scalar CMS is a headless content management system designed for speed, clarity, flexibility, and validation. It's built for modern development workflows, static sites, dynamic apps, and everything in between.

Key features include:

  • Type-safe content models with built-in validation.
  • GraphQL and REST APIs for flexible data access.
  • Real-time collaborative editing for content teams.
  • Multi-environment deployments and role-based permissions.

Quick Example

Here's how easy it is to get started with Scalar:

quickstart.ts
import { createClient } from "@scalar/sdk"; const client = createClient( { projectId: process.env.SCALAR_PROJECT_ID, apiKey: process.env.SCALAR_API_KEY, }); // fetch your content const posts = await client.query( { model: "blogPost", include: ["author", "categories"], });

Core Components

Scalar CMS has several core parts:

Scalar Core

The data and API layer: content models, validation, database handling, and runtime generation. This is the foundation of your Scalar CMS setup, handling all the backend operations and data management.

Scalar Studio

A lightweight admin UI for managing content with live preview and model editing. The visual interface where content creators and editors can manage your content efficiently with real-time previews.

Scalar CLI

A command-line interface to scaffold content models, sync schemas, run migrations, and deploy. Built for developers who prefer automation over GUI-based programming. Run scalar init to get started.

Installation

  1. 1Install Scalar CMS. Run the installation command: npx create-scalar@latest in your project directory.
  2. 2Configure your setup. Follow the prompts to choose your database (PostgreSQL, SQLite, etc.), generate models, and launch the admin panel.
  3. 3Launch your CMS. Once installation completes, start your development server with npm run dev. Your Scalar CMS admin panel will be available at http://localhost:3000/admin.

First-Time Setup

The setup wizard walks you through the initial configuration process: connecting to your chosen database, defining your first content models, and setting up user permissions. Configuring the admin panel takes only a few minutes.

Framework Integration

Scalar works seamlessly with your favorite frameworks:

Next.js React Svelte app.tsx
import { scalar } from "@scalar/next"; export default async function Page() { const posts = await scalar.getMany("blogPost"); return posts.map((p) => <Post key= { p.id} { ...p} />); }

Content Models

Define your content structure with TypeScript-like syntax:

models/blogPost.ts
export const BlogPost = { title: { type: "string", required: true } , slug: { type: "slug", from: "title" } , excerpt: { type: "text", maxLength: 280 } , body: { type: "richText" } , author: { type: "reference", to: "Author" } , publishedAt: { type: "datetime" } , };

Creating Content

Use the API to create and manage content programmatically:

create.ts
// create a new blog post const post = await client.create( { model: "blogPost", data: { title: "Hello, Scalar", body: "Your first post.", author: { connect: "author_42" } , }, } );

GraphQL API

Access your content using GraphQL:

query.graphql
query GetPosts { blogPosts(first: 10, orderBy: { publishedAt: DESC }) { id title slug excerpt author { name avatar } } }
