# PROTOCOL — TAILWIND PLUS API DOCUMENTATION TEMPLATE — STUDY / CLONE

> A SELF-CONTAINED, PIXEL-FAITHFUL REPRODUCTION OF THE TAILWIND PLUS "PROTOCOL" API-DOCUMENTATION TEMPLATE, REBUILT AS PLAIN HTML + CSS + VANILLA JS WITH NO BUILD STEP. EVERY PAGE, SECTION, HOVER STATE, AND SCROLL/ENTRANCE BEHAVIOUR IS RECREATED FROM RECON ARTIFACTS. ALL ASSETS (FONTS, COMPILED STYLESHEET, ICONS) ARE VENDORED LOCALLY SO THE CLONE RUNS OFFLINE. THIS IS A STUDY OF SOMEONE ELSE'S DESIGN, BUILT FOR LEARNING — ALL CREDIT FOR THE ORIGINAL GOES TO TAILWIND PLUS (TAILWIND LABS).

REFERENCE: `https://tailwindcss.com/plus/templates/protocol/preview`

## SUMMARY

PROTOCOL IS A PREMIUM API-REFERENCE / DEVELOPER-DOCUMENTATION TEMPLATE FROM TAILWIND PLUS. IT PRESENTS A FICTIONAL MESSAGING-PLATFORM API ("USE THE PROTOCOL API TO ACCESS CONTACTS, CONVERSATIONS, GROUP MESSAGES, AND MORE") AS A CLASSIC THREE-ZONE DOCS LAYOUT: A FIXED LEFT SIDEBAR NAVIGATION (GUIDES + RESOURCES SECTIONS WITH AN ANIMATED ACTIVE-LINK HIGHLIGHT), A STICKY FROSTED TOP BAR (LOGO, COMMAND-K SEARCH PILL, TOP NAV, THEME TOGGLE, SIGN-IN BUTTON), AND A CENTERED `prose` CONTENT COLUMN. RESOURCE PAGES SPLIT INTO A TWO-COLUMN BODY: PROSE DOCS ON THE LEFT AND STICKY DARK CODE PANELS (TABBED cURL / JAVASCRIPT / PYTHON / PHP REQUEST + RESPONSE SAMPLES) ON THE RIGHT.

THE LIVE TEMPLATE IS A NEXT.JS APP SERVED FROM `protocol.tailwindui.com`. THIS CLONE PRESERVES ITS COMPILED TAILWIND CSS V4 STYLESHEET (OBFUSCATED UTILITY CLASSES, OKLCH ZINC RAMP, EMERALD ACCENT) AND ITS EXACT RENDERED MARKUP, SELF-HOSTS ITS FONTS, AND REPLACES THE PROPRIETARY HEADLESS-UI + NEXT + FRAMER-MOTION RUNTIME WITH A SMALL VANILLA-JS SHIM THAT REIMPLEMENTS THE SAME BEHAVIOURS (THEME TOGGLE WITH `localStorage`, MOBILE NAV DRAWER, SEARCH-MODAL OVERLAY, SIDEBAR SCROLL-SPY WITH THE SLIDING HIGHLIGHT PILL, ON-THIS-PAGE TOC, CODE-PANEL TABS, COPY-BUTTON, "WAS THIS PAGE HELPFUL?" TOGGLE).

THE BUILD SPANS 12 PAGES — THE **INTRODUCTION** HOME, THREE GUIDE PAGES (**QUICKSTART**, **SDKS**, **AUTHENTICATION**, **PAGINATION**, **ERRORS**, **WEBHOOKS**), AND FIVE API-RESOURCE PAGES (**CONTACTS**, **CONVERSATIONS**, **MESSAGES**, **GROUPS**, **ATTACHMENTS**).

### SPACING / SHAPE

- LAYOUT: FIXED LEFT SIDEBAR `w-72` (18REM) / `xl:w-80` (20REM); STICKY TOP BAR `h-14`; CENTERED `max-w-2xl` PROSE STEPPING TO `lg:max-w-5xl`; TWO-COLUMN RESOURCE BODY VIA `prose` + STICKY CODE COLUMN
- SOFT HAIRLINE DIVIDERS, MINIMAL SHADOWS (CODE PANELS `shadow-md`, SEARCH MODAL `shadow-xl`)

### ANIMATION / EASINGS / INTERACTION
- DEFAULT TAILWIND TRANSITION `.15S CUBIC-BEZIER(.4,0,.2,1)`; HOVER COLOR SHIFTS ON NAV LINKS (ZINC-600 → ZINC-900 / DARK ZINC-400 → WHITE), SIDEBAR LINKS, "READ MORE" LINKS (EMERALD-500 → 600), BUTTONS, SOCIAL ICONS
- **THEME TOGGLE** — SUN/MOON ICON BUTTON FLIPS `class="dark"` ON `<html>`, PERSISTS TO `localStorage["theme"]`, RESPECTS `prefers-color-scheme` ON FIRST LOAD (INLINE BOOT SCRIPT TO AVOID FLASH)
- **SIDEBAR ACTIVE HIGHLIGHT** — A SOFT `bg-zinc-800/2.5` PILL AND AN EMERALD-500 LEFT BAR MARK THE ACTIVE LINK; THE PILL SLIDES (FRAMER-MOTION-STYLE) AS THE ACTIVE / IN-VIEW SECTION CHANGES; SUB-LINKS (`#guides`, `#resources`, AND RESOURCE-PAGE OPERATIONS) REVEAL UNDER THE ACTIVE TOP LINK
- **SEARCH MODAL** — CLICKING THE "FIND SOMETHING…" PILL (OR `Ctrl/⌘ K`) OPENS A CENTERED FROSTED DIALOG WITH A BACKDROP; ESC / OUTSIDE-CLICK / BACKDROP CLOSES (DECORATIVE SEARCH FIELD)
- **MOBILE NAV** — HAMBURGER OPENS A SLIDE-OVER DRAWER WITH THE FULL SIDEBAR NAV BELOW `lg`; BACKDROP + ESC CLOSE
- **CODE PANELS (RESOURCE PAGES)** — TABBED `role=tablist` (cURL / JAVASCRIPT / PYTHON / PHP); CLICKING A TAB MOVES THE EMERALD UNDERLINE AND SWAPS THE PANEL; A COPY BUTTON COPIES THE SNIPPET; PANELS ARE `xl:sticky xl:top-24`
- **ANCHOR HEADINGS** — HOVERING A `prose` H2/H3 REVEALS A LINK-ICON ANCHOR TO ITS LEFT (`opacity-0 → 100`)
- **WAS THIS PAGE HELPFUL?** — YES/NO SEGMENTED TOGGLE; CLICKING CROSS-FADES TO A "THANKS FOR YOUR FEEDBACK!" MESSAGE
- **HERO BACKGROUND** — RADIAL-MASKED SKEWED GRID-PATTERN SVG WITH AN EMERALD→LIME GRADIENT, FADED BLUR GLOW (HOME ONLY)

## LAYOUT & STRUCTURE

### SHARED CHROME
- **LEFT SIDEBAR (`lg+`)** — FIXED `w-72`/`xl:w-80`, BORDER-RIGHT HAIRLINE; LOGO AT TOP; THEN THE NAV: **GUIDES** (Introduction [+ Guides / Resources sub-links], Quickstart, SDKs, Authentication, Pagination, Errors, Webhooks) and **RESOURCES** (Contacts, Conversations, Messages, Groups, Attachments). ACTIVE LINK GETS THE SLIDING HIGHLIGHT PILL + EMERALD LEFT BAR.
- **TOP BAR** — STICKY, FROSTED (`bg-white/50 backdrop-blur` / dark `bg-zinc-900/20`), `h-14`, BOTTOM HAIRLINE ON SCROLL: LEFT = SEARCH PILL ("Find something…" + `Ctrl K`); RIGHT = TOP NAV (`API · Documentation · Support`), DIVIDER, THEME TOGGLE, "Sign in" PILL. BELOW `lg`: HAMBURGER + LOGO + SEARCH ICON + THEME TOGGLE.

### PAGES
- **INTRODUCTION / HOME (`index`)** — MASKED EMERALD→LIME HERO GRID BACKGROUND; H1 "API Documentation" + LEAD; QUICKSTART / EXPLORE-SDKS BUTTON PAIR; **GETTING STARTED** SECTION; **GUIDES** 4-CARD GRID (Authentication / Pagination / Errors / Webhooks, each "Read more →"); **RESOURCES** 4-CARD GRID WITH ICONS (Contacts / Conversations / Messages / Groups); "Was this page helpful?"; NEXT → Quickstart; FOOTER (copyright + X / GitHub / Discord social icons).
- **GUIDE PAGES** — `quickstart` (Choose your client, Making your first API request, What's next — with dark code blocks), `sdks` (official + community libraries), `authentication` (Basic auth / OAuth2 bearer token / Using an SDK — with request code blocks). ALL CENTERED `prose` WITH PREV/NEXT FOOTER PAGER.
- **TOP-LEVEL GUIDE PAGES** — `pagination`, `errors`, `webhooks` — `prose` DOCS WITH CODE BLOCKS, PROPERTY LISTS, AND PREV/NEXT PAGER.
- **API-RESOURCE PAGES (`contacts`, `conversations`, `messages`, `groups`, `attachments`)** — TWO-COLUMN: LEFT `prose` (resource intro, "The <model> model" property list with `<Property name type>` rows, then per-operation sections: List / Create / Retrieve / Update / Delete with "Optional attributes"), RIGHT STICKY DARK CODE PANELS (TABBED cURL/JS/Python/PHP **Request** + **Response** for each operation). EACH OPERATION HAS AN HTTP-METHOD PILL (GET/POST/PUT/DELETE) AND ENDPOINT PATH. PREV/NEXT PAGER + "Was this page helpful?" + FOOTER.

### FOOTER (ALL PAGES)
- PREV / NEXT PAGER PILLS (top-right next, top-left prev where applicable); HAIRLINE; "© Copyright 2026. All rights reserved." + X / GitHub / Discord SOCIAL ICON LINKS.
