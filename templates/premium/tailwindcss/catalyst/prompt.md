> A SELF-CONTAINED, PIXEL-FAITHFUL REPRODUCTION OF THE TAILWIND PLUS "CATALYST" APPLICATION UI KIT DEMO (THE FICTIONAL EVENT-MANAGEMENT ADMIN APP), REBUILT AS PLAIN HTML + CSS + VANILLA JS WITH NO BUILD STEP. EVERY ROUTE, EVERY PANEL, EVERY HOVER STATE, THE ACCOUNT + TEAM DROPDOWN MENUS, THE MOBILE SLIDE-IN SIDEBAR, AND THE LIGHT/DARK COLOR SCHEMES ARE RECREATED FROM RECON ARTIFACTS. THE COMPILED TAILWIND V4 STYLESHEET IS PRESERVED AND THE INTER VARIABLE FONT, EVERY EVENT PHOTO, THE TEAM LOGO, THE USER AVATAR, AND THE COUNTRY FLAG ARE VENDORED LOCALLY SO THE CLONE RUNS OFFLINE. REFERENCE: `https://catalyst-demo.tailwindui.com`

## SUMMARY

CATALYST IS TAILWIND PLUS'S MODERN REACT/UI APPLICATION KIT. ITS PUBLIC DEMO IS A FICTIONAL EVENT-TICKETING ADMIN DASHBOARD FOR AN ORGANIZATION CALLED "CATALYST", SIGNED IN AS "ERICA". THE LIVE TEMPLATE IS A NEXT.JS + HEADLESS UI + TAILWIND V4 APP SERVED FROM `catalyst-demo.tailwindui.com`; THIS CLONE PRESERVES ITS COMPILED TAILWIND CSS (UTILITY CLASSES, ZINC NEUTRAL RAMP IN OKLCH/LAB, BLUE-500 FOCUS ACCENT, BUILT-IN DARK MODE), ITS EXACT RENDERED MARKUP AND COPY, SELF-HOSTS THE INTER VARIABLE WOFF2, VENDORS ALL IMAGERY LOCALLY, AND REPLACES THE NEXT/HEADLESS-UI RUNTIME WITH A SMALL VANILLA-JS LAYER THAT REIMPLEMENTS THE SAME BEHAVIOURS (ACCOUNT DROPDOWN, TEAM SWITCHER DROPDOWN, MOBILE SLIDE-IN SIDEBAR WITH BACKDROP, DARK-MODE TOGGLE VIA `prefers-color-scheme`, KEYBOARD/OUTSIDE-CLICK DISMISS).

THE BUILD SPANS 37 PAGES THAT FALL INTO TWO LAYOUT FAMILIES — THE **SIDEBAR APPLICATION LAYOUT** (FIXED 16REM SIDEBAR, ROUNDED WHITE CONTENT PANEL ON A ZINC-100 PAGE, MOBILE TOP NAVBAR) USED BY: THE **HOME** DASHBOARD, **EVENTS** INDEX, FOUR **EVENT DETAIL** PAGES (1000–1003), **ORDERS** INDEX, TWENTY-SIX **ORDER DETAIL** PAGES (3000–3025), AND **SETTINGS**; PLUS THE **CENTERED AUTH LAYOUT** (NO SIDEBAR, CARD ON A ZINC-50/WHITE GRADIENT FIELD) USED BY: **LOGIN**, **REGISTER**, AND **FORGOT PASSWORD**.

### ANIMATION / EASINGS / INTERACTION
- **ACCOUNT DROPDOWN** (SIDEBAR FOOTER — ERICA AVATAR + EMAIL) — CLICK OPENS A `role=menu` POPOVER ANCHORED TOP-START ABOVE THE BUTTON: **MY ACCOUNT · PRIVACY POLICY · SHARE FEEDBACK · ─── · SIGN OUT**. ITEMS HIGHLIGHT `bg-blue-500 text-white` ON FOCUS/HOVER. FADE/SCALE-IN (`data-closed:scale-95 opacity-0`), ESC / OUTSIDE-CLICK CLOSES.
- **TEAM SWITCHER DROPDOWN** (SIDEBAR HEADER — "CATALYST") — CLICK OPENS A MENU: **SETTINGS(GEAR) · ─── · CATALYST(CHECK, CURRENT) · BIG EVENTS(BE AVATAR) · ─── · NEW TEAM…(PLUS)**, ANCHORED BELOW THE HEADER.
- **MOBILE SLIDE-IN SIDEBAR** — BELOW `lg`, A TOP NAVBAR WITH A HAMBURGER REPLACES THE FIXED SIDEBAR; TAPPING IT SLIDES THE SAME SIDEBAR IN FROM THE LEFT (`translate-x` TRANSITION) OVER A `bg-black/30` BACKDROP, WITH AN X CLOSE BUTTON; BACKDROP / X / ESC CLOSES.
- **NAV + LIST HOVER** — NAV ITEMS, TABLE ROWS, AND EVENT/ORDER LIST ROWS GET `hover:bg-zinc-950/5` (DARK: `white/5`); WHOLE TABLE ROWS ARE CLICKABLE (OVERLAID ANCHORS) AND HOVER-HIGHLIGHT.
- **BUTTONS** — SOLID `bg-zinc-950 text-white hover:bg-zinc-800` (PRIMARY); OUTLINE `ring-1 ring-zinc-950/10 hover:bg-zinc-950/2.5`; ALL `rounded-lg`, SUBTLE PRESS STATE.
- **INPUTS** — `rounded-lg` WITH ZINC RING, BLUE-500 FOCUS RING; CHECKBOX/SWITCH, FLOATING SELECTS WITH CHEVRON; FORMS ARE CLIENT-SIDE ONLY (NO SUBMIT NAVIGATION).
- **DARK MODE** — THE COMPILED CSS SHIPS A FULL `dark:` VARIANT KEYED ON `prefers-color-scheme`; THE CLONE HONORS IT AND PROVIDES A MANUAL TOGGLE THAT FLIPS A `dark` CLASS ON `<html>`.

## LAYOUT & STRUCTURE

### SHARED CHROME — SIDEBAR APPLICATION LAYOUT
- **SIDEBAR (≥lg, FIXED LEFT, w-64)** — HEADER: TEAM SWITCHER BUTTON ("CATALYST" LOGO + WORDMARK + CHEVRON → TEAM DROPDOWN). BODY: PRIMARY NAV (HOME · EVENTS · ORDERS · SETTINGS, EACH WITH HERO-ICON), THEN "UPCOMING EVENTS" SECTION LINKING THE FOUR EVENTS, THEN A SPACER + "SUPPORT" / "CHANGELOG" SECONDARY LINKS. - **MOBILE NAVBAR (<lg)** — HAMBURGER (OPENS SLIDE-IN SIDEBAR) + SPACER + ACCOUNT AVATAR MENU.

### PAGES — SIDEBAR LAYOUT
1. **HOME (`/`)** — GREETING "Good afternoon, Erica"; PERIOD FILTER ("Last week"); FOUR KPI STAT CARDS (TOTAL REVENUE $2.6M +4.5%, AVERAGE ORDER VALUE $455 −0.5%, TICKETS SOLD 5,888 +4.5%, PAGEVIEWS 823,067 +21.2%) EACH WITH A TOP HAIRLINE; "RECENT ORDERS" TABLE (ORDER NUMBER · PURCHASE DATE · CUSTOMER · EVENT(THUMB+NAME) · AMOUNT) OF 10 ROWS, ROWS LINK TO `/orders/:id`.
2. **EVENTS (`/events`)** — "Events" TITLE + SEARCH INPUT + "SORT BY" SELECT + "CREATE EVENT" BUTTON; A LIST OF FOUR EVENTS, EACH ROW = COVER IMAGE + NAME(LINK) + DATE/TIME · VENUE + "N/M TICKETS SOLD" + STATUS BADGE (ON SALE / CLOSED) + OVERFLOW MENU. EVENTS: BEAR HUG: LIVE IN CONCERT (May 20, 350/500, On Sale), SIX FINGERS — DJ SET (Jun 2, 72/150, On Sale), WE ALL LOOK THE SAME (Aug 5, 275/275, Closed), VIKING PEOPLE (Dec 31, 6/40, On Sale).
3. **EVENT DETAIL (`/events/1000`–`/events/1003`)** — BREADCRUMB "← Events"; COVER + TITLE + STATUS BADGE + DATE/VENUE; "EDIT" + "VIEW" BUTTONS; THREE KPI CARDS (TOTAL REVENUE, TICKETS SOLD, PAGEVIEWS); "RECENT ORDERS" TABLE FILTERED TO THAT EVENT (ORDER NUMBER · DATE · CUSTOMER · AMOUNT).
4. **ORDERS (`/orders`)** — "Orders" TITLE; A TABLE OF ALL 26 ORDERS (ORDER NUMBER · PURCHASE DATE · CUSTOMER · EVENT(THUMB+NAME) · AMOUNT), EACH ROW LINKING TO ITS ORDER DETAIL.
5. **ORDER DETAIL (`/orders/3000`–`/orders/3025`)** — BREADCRUMB "← Orders"; "Order #NNNN" + "Successful" BADGE; META ROW (AMOUNT · CARD BRAND ••••NNNN · DATE) + "REFUND" / "RESEND INVOICE" BUTTONS; "SUMMARY" DEFINITION LIST (CUSTOMER, EVENT(THUMB LINK), AMOUNT, AMOUNT AFTER EXCHANGE RATE, FEE, NET); "PAYMENT METHOD" DEFINITION LIST (TRANSACTION ID, CARD NUMBER, CARD TYPE, EXPIRY, OWNER, EMAIL, ADDRESS, COUNTRY(FLAG), CVC "Passed successfully").
6. **SETTINGS (`/settings`)** — "Settings" FORM WITH SECTIONS (ORGANIZATION NAME, ORGANIZATION BIO(TEXTAREA), ORGANIZATION EMAIL + "SHOW ON PUBLIC PROFILE" SWITCH, ADDRESS(STREET/CITY/REGION SELECT/POSTCODE/COUNTRY SELECT WITH FLAG), CURRENCY SELECT), HAIRLINE-SEPARATED, "RESET" + "SAVE CHANGES" BUTTONS.

### PAGES — CENTERED AUTH LAYOUT (NO SIDEBAR)
7. **LOGIN (`/login`)** — LOGO; "Sign in to your account"; EMAIL + PASSWORD FIELDS; "REMEMBER ME" CHECKBOX + "FORGOT PASSWORD?" LINK; "LOGIN" BUTTON; "Don't have an account? Sign up".
8. **REGISTER (`/register`)** — LOGO; "Create your account"; EMAIL + FULL NAME + PASSWORD + COUNTRY SELECT (Canada/Mexico/United States) + "GET EMAILS…" CHECKBOX; "CREATE ACCOUNT" BUTTON; "Already have an account? Sign in".
9. **FORGOT PASSWORD (`/forgot-password`)** — LOGO; "Reset your password" + INSTRUCTIONS; EMAIL FIELD; "RESET PASSWORD" BUTTON; "Don't have an account? Sign up".
