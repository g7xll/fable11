## THE STACKING-CARDS MECHANIC (EXACT BEHAVIOR — THIS IS THE POINT)

THE STACK COMBINES PURE-CSS STICKY PINNING WITH A SCROLL-LINKED JS TRANSFORM. KEY DETAILS:

- EACH CARD WRAPPER IS A TALL SECTION (`min-height: 120vh`) SO THERE IS TRAVEL BETWEEN CARDS. THE CARD ITSELF IS `position: sticky`.
- STICKY OFFSETS INCREASE PER CARD SO THE STACK FANS DOWNWARD: CARD `n` PINS AT `top: calc(var(--nav) + n * 22px)`. THIS LEAVES A ~22PX STRIP OF EACH UNDERLYING CARD VISIBLE AT THE TOP — THE DECK EDGE.
- THE CARD HAS A LARGE BORDER RADIUS (`28px`), A STRONG SOFT SHADOW (`0 -10px 40px rgba(0,0,0,0.18), 0 30px 60px rgba(0,0,0,0.28)`) SO STACKED EDGES CAST ONTO EACH OTHER, AND `overflow: hidden`.
- A SCROLL-LINKED EFFECT (JS, VIA `IntersectionObserver` + A SINGLE `scroll`/`requestAnimationFrame` LOOP) COMPUTES, FOR EACH CARD, HOW FAR THE **NEXT** CARD HAS COVERED IT (0 → 1) AND APPLIES TO THE COVERED CARD: `transform: scale(1 - 0.06 * p)` AND `filter: brightness(1 - 0.18 * p)` PLUS A DARKENING OVERLAY `opacity: 0.5 * p`. SO AS A CARD IS BURIED IT SHRINKS A TOUCH, DARKENS, AND ITS CONTENT DIMS — READING AS DEPTH. EASING ON `p` IS `easeOutCubic`.
- INCOMING CARDS ALSO GET A TINY ENTRANCE: AS A CARD CROSSES INTO VIEW IT ANIMATES FROM `translateY(40px)` / `opacity 0.6` TO REST USING THE SAME SCROLL PROGRESS (NOT A ONE-SHOT) SO IT FEELS PHYSICALLY LINKED TO SCROLL, NOT TIME.
- RESPECT `prefers-reduced-motion`: DISABLE THE SCALE/BRIGHTNESS SCRUB AND KEEP ONLY THE STATIC STICKY STACK.

### CORE CSS (THE LOAD-BEARING PART)

```css
:root { --nav: 84px; --stack-step: 22px; }

.deck { position: relative; }

.card-section {            /* one per project — gives scroll distance */
  min-height: 120vh;
  display: flex;
  justify-content: center;
}

.card {
  position: sticky;
  /* top is set per-card inline: top: calc(var(--nav) + index * var(--stack-step)) */
  width: min(1100px, 92vw);
  border-radius: 28px;
  overflow: hidden;
  box-shadow: 0 -10px 40px rgba(0,0,0,.18), 0 30px 60px rgba(0,0,0,.28);
  transform-origin: 50% 0%;        /* shrink toward the pinned top edge */
  will-change: transform, filter;
}

.card .cover {                      /* darkening overlay driven by JS --p */
  position: absolute; inset: 0;
  background: #000; opacity: 0; pointer-events: none;
}

@media (prefers-reduced-motion: reduce) {
  .card { transform: none !important; filter: none !important; }
}
```

### CORE JS (SCROLL-LINKED SCRUB)

```js
const cards = [...document.querySelectorAll('.card')];
const easeOutCubic = t => 1 - Math.pow(1 - t, 3);

function frame() {
  const navH = 84, step = 22, vh = innerHeight;
  cards.forEach((card, i) => {
    const next = cards[i + 1];
    if (!next) { card.style.transform = ''; return; }
    const pinTop = navH + i * step;
    const nextRect = next.getBoundingClientRect();
    // progress 0..1 as the NEXT card travels from bottom of viewport up to the pinned card's top
    const start = vh;                       // next card enters from here
    const end = pinTop + step;              // covered when next reaches the deck edge
    let p = (start - nextRect.top) / (start - end);
    p = easeOutCubic(Math.min(1, Math.max(0, p)));
    card.style.transform = `scale(${1 - 0.06 * p})`;
    card.style.filter = `brightness(${1 - 0.18 * p})`;
    card.querySelector('.cover').style.opacity = (0.5 * p).toFixed(3);
  });
  ticking = false;
}
let ticking = false;
addEventListener('scroll', () => { if (!ticking) { ticking = true; requestAnimationFrame(frame); } }, { passive: true });
addEventListener('resize', frame); frame();
```
